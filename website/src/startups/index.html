In practice
Same job. Two different architectures.
Turn bank statement PDFs into a clean Excel workbook.
AI as a feature
SimpleBooks

User uploads a PDF. The app sends it to Claude via API. Claude extracts the transactions. The app runs Python to format a workbook. The user downloads the result.

Works great. Until:

  • The user wants a new column
  • One client needs a different format
  • A bank changes their PDF layout

Each one is a feature request. Each one waits for an engineer.

AI-native
SmartBooks

User drops the PDF into a chat. Claude Code reads it, runs the app's Python scripts, and produces a workbook.

When something changes:

  • New column? The user says so. Done.
  • Different format? Claude Code edits the script.
  • New PDF layout? The AI adapts on the spot.

No feature request. No engineer. No wait.
